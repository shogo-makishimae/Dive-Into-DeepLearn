# 控制流求梯度

## 什么是控制流求梯度？

先将控制流求梯度拆开看（控制流+梯度）

1. 控制流：if/which/for语句，给程序引导（控制）执行方向的；
2. 求梯度（见梯度）：自动求梯度即可；

控制流既然是引导语句，会产生分支结构，计算机关于分支结构出现了两种流派（动态计算图/静态计算图）

### 静态计算图

特点：静态图的核心思想是“先盖房子，再住人”（先编译，后执行）。

优点：计算速度块，内存更优化

缺点：难以直观调试

下图为输入偶数时走的是$y=x^2$的这个分支，输入为奇数时$y=x$的这个分支黑色的

![Dynamic1.png](Dive-Into-DeepLearn\introduction\imges\Gradient\Dynamic1.png "静态计算图1")

下图为输入为奇数时走的是$y=x$的这个分支，输入为偶数时$y=x^2$的这个分支是黑色的

![Dynamic2.png](Dive-Into-DeepLearn\introduction\imges\Gradient\Dynamic2.png "静态计算图2")

说明：静态图的运行逻辑

1. 在静态图中（比如构建 TensorFlow 1.x 的图时），你写的 `y = x^2` 只是在向编译器画一张“蓝图”。此时的 `x` 和 `y` 里面根本没有数字，它们只是**符号张量（Symbolic Tensor/Placeholder）**。如果你在这个时候 `print(y)`，你看到的不是数字 4，而是一段类似 `<tf.Tensor 'pow:0' shape=(1,) dtype=float32>` 的字符串。
2. **真正的黑盒运行：** 等到图全部画完，你调用 `session.run()` 或者传入真实数据开始跑的时候，整个图会被丢进底层的 C++ 引擎（甚至 GPU 硬件）中高速运行。在这个高速运行的黑盒里，为了极致省显存，编译器会**疯狂地复用存储空间，把不需要用来算梯度的中间结果直接覆盖掉**。由于运行在 C++ 底层，Python 的 `print` 和断点根本插不进去，一旦报错，抛出的往往是底层节点图崩溃的错综复杂的错误堆栈。

**代表框架：** TensorFlow 1.x, JAX (搭配 `@jit`), PyTorch 的 `torch.compile` 或导出 ONNX
**核心逻辑：** 先编译出完整的计算拓扑图，再灌入数据执行。

- **优点（为什么工业界离不开）：**
  - **极致的运行速度：** 编译器一旦拿到了完整的图，就可以进行大量的数学化简和底层硬件（GPU/TPU）级别的极致优化。
  - **内存优化更佳：** 能做到最优的内存复用和显存管理。
  - **易于跨平台部署：** 静态图可以被序列化（保存成一个纯粹的数据结构文件，比如 `.pb` 或 `.onnx`）。C++、Java、Rust 或者端侧芯片都可以直接读取这个文件并执行，完全不需要 Python。
- **缺点：**
  - **反人类的调试体验：** 一旦图被编译，它就变成了一个“黑盒”。如果运行出错，报出的错误往往是底层 C++ 级别的图节点错误，很难追踪到你当初写的 Python 代码。你也无法在编译后的图里随便插入 `print`。
  - **灵活性差：** 写复杂的控制流非常痛苦，必须使用框架规定的特殊算子（比如 `tf.while_loop` 或 `jax.lax.cond`），语法很绕。

### 动态计算图

特点：动态图的核心思想是“所见即所得，边跑边建图”。

优点：便于调试（可以插入 `print`观察输出结果），灵活性高;

缺点：运算速度慢，部署困难

下图为输入偶数时：只有偶数分支

![Non-Dynamic1.png](Dive-Into-DeepLearn\introduction\imges\Gradient\Non-Dynamic1.png "动态计算图1")

下图为输入奇数时：只有奇数分支

![Non-Dynamic2.png](Dive-Into-DeepLearn\introduction\imges\Gradient\Non-Dynamic2.png "动态计算图2")

说明：动态图的生成逻辑

只有偶数（奇数）的分支，为什么会只有偶数的分支，关于动态图的生成逻辑，动态图是边跑边画图的，不知道这个图是什么样子的，只能计算完当前节点时才画图，而此时插入 `print` 语句就可以直接输出，方便调试

总结：在动态图中，复杂的控制流（`if/else`, `while`）在每次前向传播时，都会被“拉平”成一条**纯粹的、没有任何分支的、线性的数学运算路径**。反向传播只需要沿着这条被拉平的“一次性算术题”往回算就可以了。这就叫 Define-by-run（运行即定义）

```python
import torch
x = torch.tensor([1.0, 2.0, 3.0], requires_grad=True)
y1 = x**2
y2 = x
for i in range(len(x)):
    # 判断元素奇偶时应使用标量值
    if int(x[i].item()) % 2 == 0:
        print(y1)
        # 每次反向传播前清零/重置梯度，避免累加
        x.grad = None
        y1.backward(torch.ones_like(x))
        print(x.grad)
    else:
        print(y2)
        x.grad = None
        y2.backward(torch.ones_like(x))
        print(x.grad)
```

说明：

1. `x[i].item()` 与 `x[i]` 的区别：前者是个标量（数），后者是张量tensour；
2. 运行错误：第一次写的时候输出是 `tensor[1,1,1]，tensor[3,5,7]，tensor[4,6,8]` 其实观察可得3=1+2，5=1+4，7=1+6，4=3+1，6=5+1，8=7+1，红色的数字是正确结果，和红色相加的数字是之前结果的类加，问题出在没有对梯度进行清零，在每次计算后补 `x.grad = None` ，得出现在正常的结果，输出为 `tensor[1,1,1]，tensor[2,4,6]，tensor[1,1,1]`；
3. 逻辑错误：上述代码不是动态计算图，为什么不是呢，因为 `y1 = x**2，y2 = x`
   代码逻辑是：先做完全部两套计算，再用 `if` 决定拿哪套结果去反向求导，正确的逻辑应该是边算边画图，最后一次性反向求导

动态图计算：第一版

```python
import torch

x = torch.tensor([1.0, 2.0, 3.0], requires_grad=True)

# 创建一个空的 y 用来存放结果，保留梯度追踪能力
y = torch.zeros_like(x) # y = [y[0] = 0, y[1] = 0, y[2] = 0]

# 【前向传播阶段】：控制流在这里发挥作用，动态建图
for i in range(len(x)):
    if int(x[i].item()) % 2 == 0:
        y[i] = x[i] ** 2  # 如果是偶数，为 y[i] 建立平方的计算图分支
    else:
        y[i] = x[i]       # 如果是奇数，为 y[i] 建立线性映射的计算图分支

print("前向传播计算结果 y:", y) 
# 输出应为 tensor([1., 4., 3.], grad_fn=<CopySlices>)

# 【反向传播阶段】：沿着刚才动态建立的、高低起伏的路径一次性回传
y.backward(torch.ones_like(x)) # 等价于 y.backward(torch.tensor([1.0, 1.0, 1.0]))

print("反向传播求导结果 x.grad:", x.grad) 
# 输出应为 tensor([1., 4., 1.])
# 解释：
# x[0]=1 (奇数)，导数是 1
# x[1]=2 (偶数)，导数是 2*x = 4
# x[2]=3 (奇数)，导数是 1
```

说明：既然是动态图计算，在输入之前不知道输入结果，所以将计算过程放到if语句中，进入if语句中在计算，y用来存放计算后的结果，输出结果 `tensor([1., 4., 3.], grad_fn=<CopySlices>)` ，这个是计算后的结果，然后用这个y反向传播求梯度，即在这个基础上求梯度即可 `y.backward(torch.ones_like(x))` 要和x的格式相同（标量都是标量，张量都是张量），结果是 `tensor([1., 4., 1.])` ；

上述代码，逻辑是对的，不过现在有更简单的写法

1. 关于赋值操作双for循环太冗长，`y[i] = x` 即可；
2. 关于反向传播阶段，双for循环操作每个元素有些冗长，操作行，操作行是不是直接回到了最初如何对张量进行求梯度，求和再反向传播， `y.sum(dim=1)[0].backward(retain_graph=True)` /`y_sum = y.sum(),y_sum.breakward`等价 `y.backward(torch.ones_like(y))`;

```python
import torch

x = torch.tensor([1.0, 2.0, 3.0], requires_grad=True)

# 创建一个空的 y 用来存放结果，保留梯度追踪能力
# y = torch.zeros_like(x) # y = [y[0] = 0, y[1] = 0, y[2] = 0]
y = torch.zeros(3, 3)
print("初始 y:", y)

# 【前向传播阶段】：控制流在这里发挥作用，动态建图
for i in range(len(x)):
    if int(x[i].item()) % 2 == 0:
        y[i] = x ** 2  # 如果是偶数，为 y[i] 建立平方的计算图分支,这里是直接把 x 的值赋给 y                          # [i]，所以 y[i] 和 x 是同一个张量，具有相同的计算图

    else:
            y[i] = x # 如果是奇数，为 y[i] 建立线性映射的计算图分支,这里是直接把 x 的值赋                          # 给 y[i]，所以 y[i] 和 x 是同一个张量，具有相同的计算图
          
y.sum(dim=1)[0].backward(retain_graph=True)
print("反向传播计算结果 y:", x.grad)
x.grad = None  # 每次反向传播后清零梯度，避免累加

y.sum(dim=1)[1].backward(retain_graph=True)
print("反向传播计算结果 y:", x.grad)
x.grad = None  # 每次反向传播后清零梯度，避免累加

y.sum(dim=1)[2].backward(retain_graph=True)
print("反向传播计算结果 y:", x.grad)
x.grad = None  # 每次反向传播后清零梯度，避免累加s
```

说明：`y.sum(dim=1)[0].backward(retain_graph=True)`

1. `y.sum(dim=1)[0]` 对矩阵的第一行求和，（在numpy中的参数式esis，在torch中的参数是dim），`[0]`代表第0行；
2. `backward()` 需要说明一点，这个函数只能对标量进行求梯度，我们通过求和将张量变成标量
3. `retain_graph=True` ：保存图选择true，这里的图指的就是计算图，`backward()` 默认是求导后图“阅后即焚”，这个代码中如果不保留，第二次求导就会报错，我们常识可知，求导都是一层一层的求，如果不保存上次的结果，下次求导必然报错；
4. 使用场景：非常罕见，这里即可能产生创新，又说明现在集成的非常完善，通常只有在计算**雅可比矩阵（Jacobian）**，或者需要分别单独观察网络中某一个特定输出分支的梯度时使用；
5. 实际使用中的如何求梯度？

```python
predictions = model(inputs) # 输出可能是一个 128x1000 的大矩阵（128个样本，1000个分类）
loss = loss_function(predictions, targets) # 计算损失
loss.backward()                       # 求导
```
